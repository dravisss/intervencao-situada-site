## O que é uma perturbação

Uma **perturbação** é um movimento intencional sobre um agenciamento. Ela pode desterritorializar (desestabilizar uma configuração existente), reterritorializar (estabilizar uma configuração emergente) ou ambos simultaneamente.

Ela responde a uma tensão situada identificada a partir das narrativas dos membros do agenciamento. Se a tensão foi construída a partir da paisagem disposicional do agenciamento, a perturbação é legítima - independentemente de amplificar uma linha de fuga existente ou criar um movimento novo.

No fringir dos ovos, "perturbação" é um nome chique para intervenção. E para entender como podemos "perturbar" a ordem estabelecida de um sistema, precisamos falar sobre atratores, linhas de fuga e adjacentes possíveis.

---

## Atratores, linhas de fuga e adjacente possível

A tensão situada diz o que está presente na composição atual do sistema. A perturbação diz o que fazer nesse emaranhado.

Mas o que realmente orienta a direção? Entre a leitura da tensão e o desenho da
perturbação, há uma camada que responde a três perguntas:

- Para onde o sistema já tende a ir?
- Por onde ele já está escapando?
- O que está ao alcance agora?

### Atratores

Um atrator funciona como um poço gravitacional. O sistema pode até
se afastar de uma tendência por um tempo por causa de uma perturbação externa, uma tentativa de mudança, um empurrão. Mas se o empurrão não for forte o bastante
para tirá-lo da borda do poço, o sistema volta. 

E não é que ele volta exatamente para o mesmo lugar, ele volta para a mesma região. A mesma dinâmica. As mesmas reclamações disfarçadas de problemas diferentes.

Atratores são padrões de interação que se repetem e estabilizam um território. 

Uma reunião onde todo mundo repete as mesmas opiniões. Um conflito entre áreas que sempre termina no mesmo silêncio ensurdecedor. Um processo que todo mundo ignora mas ninguém faz nada pra mudar. Narrativas que contam histórias parecidas. Enfim, é o padrão que persiste apesar dos pesares.

O teste prático para saber se você está diante de um atrator: você já tentou mudar isso antes? Se o padrão voltou, é um atrator. Não importa se foi um novo processo, uma reestruturação ou uma ferramenta nova.

E claro, atratores não são intrinsecamente ruins. Um ritual semanal que todo mundo participa, uma decisão que é tomada informalmente sem precisar de hierarquia e um acordo que orienta decisões são atratores positivos e até mesmo desejáveis. 

O problema não é o atrator em si. É quando o sistema se aprisiona num padrão que já não serve mais aos interesses das pessoas e ao que a organização precisa naquele momento.

Aí o trabalho de intervenção não vai ser sobre tentar de novo com mais força bruta, e sim sobre entender o contorno do que mantém o sistema preso a esse padrão.

Para quebrar padrões que "atraem" a organização para direções indesejadas, precisamos conhecer os movimentos de desterritorialização do sistema.

### Linhas de fuga

Se o atrator é para onde o sistema tende ir "naturalmente", a linha de fuga é o
movimento que escapa dessa tendência quase como uma tentativa do sistema de destruir sua própria identidade.

Ela é o desvio, a exceção, a desterritorialização dos códigos, a subjetividade rebelde que escapa das caixinhas e normas estabelecidas. 

É qualquer movimento que desafia alguma premissa e perturba o território, seja com um desafio à hegemonia ou subvertendo os códigos vigentes. 

Uma perturbação pode amplificar uma linha de fuga existente ou criar uma fissura nos muros do território. 

Quando uma linha de fuga se consolida, ela não está apenas "fugindo de" alguma coisa que precisa ser destruída, ela está criando um território novo. 

O trabalho de intervenção social muitas vezes é um trabalho de identificar por onde o sistema "vaza" e amplificar essa rachadura, essa fuga ao que está dado como "normal". 

No mundo corporativo isso é frequentemente descrito como "desafiar o status quo". 

Mas é preciso ter cuidado com as linhas de fuga que escolhemos amplificar, pois dependendo da direção que ela toma, pode acabar virando uma "linha de morte" para o sistema, destruindo qualquer coisa considerada familiar.

Por isso é importante trabalhar com aquilo que chamamos de adjacente possível, para dar pequenos passos em direção a novos territórios ao invés de romper abruptamente com a identidade do sistema.


### Adjacente possível

De todas as direções que uma intervenção poderia tomar, nem todas
estão ao alcance. 

Stuart Kauffman chama de adjacente possível aquilo que é viável a partir da configuração atual do sistema. Veja bem, o adjacente não é tudo que se pode imaginar, mas o que está ao alcance agora. A pergunta que nos guia aqui é "qual é o próximo passo possível que podemos dar?"

Uma intervenção não pode levar o sistema a qualquer lugar. 

Ela só pode movê-lo para o que está adjacente. Tentar pular de "zero
autonomia" para "autogestão plena", por exemplo, é um
salto que muitas organizações não comportam.

Cada nova ideia ou intervenção revela novos territórios "adjacentes" antes desconhecidos.

- A invenção da eletricidade revelou no "mapa" os territórios da eletrônica, eletrodomésticos, iluminação, computadores, etc.
- O smartphone abriu no "mapa" os territórios dos apps, streaming, redes sociais móveis, rastreamento, etc.
- A internet expôs os territórios do comércio digital, redes sociais, compartilhamento de dados, nuvem, etc.
- A inteligência artificial está revelando os territórios de carros autônomos, diagnósticos médicos, chatbots, etc.

O adjacente possível não é algo que a gente planeja como sendo o próximo passo ideal. Ele se percebe  pelas narrativas, pelas linhas de fuga, pelos ensaios que o sistema já está fazendo. Uma perturbação bem desenhada explora o adjacente — e, ao fazê-lo, expande o leque do que se torna possível em seguida.

**Orientações para a o desenho de perturbações**

Os três conceitos formam um sistema de orientação para intervir:

- O **atrator** diz para onde o sistema tende a ir sozinho —
  e, portanto, o que a perturbação precisa deslocar, amplificar ou abafar.
- A **linha de fuga** diz por onde o sistema já está escapando
  — e, portanto, o que a perturbação pode amplificar ou estabilizar.
- O **adjacente possível** diz o que está ao alcance agora —
  e, portanto, até onde a perturbação pode ir sem forçar a barra.

Uma perturbação bem desenhada faz os três simultaneamente:
amplifica uma linha de fuga existente, desloca o atrator atual e explora o adjacente possível. Ela não precisa de um "futuro desejado" externo, uma vez que o vetor já está dado pela composição atual do sistema.

Agora, para explorar novos adjacentes, precisamos de estruturas temporárias para pavimentar o caminho. Precisamos de andaimes.

## Andaimes 

Imagine uma casa em reforma. As paredes estão nuas, o telhado foi refeito, mas as vigas ainda estão expostas. O que permite os pedreiros reformarem as paredes não são as vigas, pois elas não suportam o peso sozinhas. Para levantar as paredes utiliza-se um andaime. Ele está ali temporariamente, apoiado no chão e encostado no que já existe, para que os pedreiros possam trabalhar onde ainda não há chão firme. No dia em que a parede estiver de pé, o andaime sai.

Uma intervenção não pode esperar o sistema estar pronto para mudar. Se esperar, nunca acontece. O que ela pode fazer é colocar um andaime onde o terreno ainda não se consolidou.  Neste caso nós precisamos de uma estrutura temporária que permite ao sistema ocupar um território novo antes de ter desenvolvido a capacidade de se sustentar sozinho nele.

Você já viu uma criança aprender a andar de bicicleta.  Oras... talvez você já tenha sido essa criança. Qual é a função das rodinhas na bicicleta? Impedir que a crianç caia? Ensinar ela a ter equilíbrio?

Eu diria que não, as rodinhas não ensinam equilíbrio. Elas criam uma condição para que a criança pedale, sinta a velocidade, aprenda a olhar para frente, ganhe confiança. 
O equilíbrio vem depois, quando as rodinhas somem e a criança descobre que já está sustentando o próprio peso. As rodinhas são andaimes. Se você nunca tirar, a criança nunca aprende a se equilibrar. Se você tirar antes de ela estar pronta, ela cai e pode desistir.

A seção anterior terminou com três perguntas: para onde o sistema tende a ir, por onde ele já está escapando, o que está ao alcance agora. Atrator, linha de fuga, adjacente possível. O que faltou foi uma quarta: o que segura o sistema enquanto ele se move do atrator atual para o adjacente possível, usando a linha de fuga como caminho?

Atratores são padrões estabilizados. Linhas de fuga são desvios que escapam da ordem estabelecida. O adjacente possível é o próximo passo que a organização pode dar numa nova direção. 

A perturbação tenta deslocar o primeiro, amplificar o segundo e explorar o terceiro. Mas deslocar um atrator desestabiliza. As pessoas sentem o chão sumir. O sistema pode colapsar antes de encontrar o novo padrão. O andaime é o que evita esse colapso. É o que segura o sistema na travessia entre um atrator e outro.

Talvez você conheça uma equipe que decidiu experimentar autogestão. As pessoas estavam animadas com a ideia, mas no primeiro conflito sobre quem decide o quê, a ansiedade subiu e alguém pediu para o gestor resolver. 

O gestor poderia voltar atrás e simplesmente tomar a decisão. Em vez disso, criou um andaime: uma reunião semanal de trinta minutos onde o time discute decisões difíceis, ele está presente mas não decide. Ele pergunta "O que te impede de tomar uma decisão sobre isso?" e provoca as pessoas sobre a autoridade que elas possuem agora para tomar riscos. 

O andaime não é a autogestão. É a estrutura temporária que permite ao time aprender a decidir junto sem alguém acima resolver. Depois de alguns meses, a reunião semanal perdeu o sentido. O time já sustentava o próprio peso. O andaime foi desmontado.

O truque do andaime bem feito é que ele desaparece no momento certo. 

Nem antes, quando o sistema ainda não desenvolveu a musculatura necessária, nem depois, quando já virou dependência. Um andaime que nunca sai não é suporte para a construção, é parte da estrutura permanente. O sistema aprendeu a apoiar-se nele, não a sustentar-se sozinho. É por isso que o primeiro teste de um andaime não é "está funcionando?", mas "já podemos tirar?"

## Domínios do Cynefin® como contexto

Nem toda a intervenção vai exigir uma experimentação constante ou movimentos rápidos para apagar incêndios. A depender do tipo de problema que estamos lidando, vamos desenhar intervenções diferentes. 

Estamos lidando com um problema ordenado? Conseguimos ter um alto nível de previsibilidade sobre o que vai acontecer? É possível conhecer as relações de causalidade envolvidas? 

Ou estamos falando de um problema não-ordenado onde nem adianta tentar inferir relações de causalidade? Onde mesmo um especialista se torna pouco útil para "resolver" a questão?

Não podemos tratar todo problema da mesma forma. O tipo de perturbação adequado depende do domínio em que a tensão se encontra:

| Domínio        | Caráter da tensão                                   | Tipo de intervenção                                       |
| -------------- | --------------------------------------------------- | --------------------------------------------------------- |
| **Claro**      | Causa e efeito são óbvios para todos                | Aplicar procedimento conhecido, intervenção direta        |
| **Complicado** | Causa e efeito existem mas precisam de análise      | Especialista analisa, propõe solução, implementa          |
| **Complexo**   | Causa e efeito só são conhecidos retrospectivamente | Experimentos seguros para falhar — sonda, sente, responde |
| **Caótico**    | Sistema em colapso, ação imediata necessária        | Agir para estabilizar, depois transicionar                |

Uma mesma tensão pode ser lida em domínios diferentes por atores diferentes. E tá tudo bem. Não existe uma resposta certa sobre qual é o domínio em que um probelma se encontra. Parte da arte de definir problemas é decidir coletivamente em que domínio se está operando.

## Tipologia de intervenções

Existem muitos caminhos que podem ser tomados para intervir num sistema social. 

As perturbações vão desde mudanças nas condições materiais (como cargos, salários e equipamentos) até intervenções na linguagem, nos símbolos e nos discursos da organização.

No entanto, na tentativa de sistematizar as diferentes formas que uma intervenção pode assumir, eu recorro mais uma vez a um autor bastante relevante na área de complexidade aplicada.

Dave Snowden, no artigo [Updated Estuarine typologies](https://thecynefin.co/updated-estuarine-typologies/) propõe nove tipos de micro-intervenções, organizadas em três grupos.

Eu apresento aqui uma leitura sobre a tipologia, com foco na compreensão didática de diferentes tipos de intervenção. 

Mais uma vez, peço que não considerem isso como uma categorização estática e sim como diferentes lentes que podemos usar para desenhar perturbações. Também não é um manual ou um guia estrito a ser seguido. Use a tipologia para refletir sobre diferentes intervenções que você pode fazer para uma mesma tensão identificada.

### Ações Diretas

#### Bússola
Altera custos de energia e tempo — aumenta ou diminui o esforço necessário para algo se mover.

*Exemplo*: A política de aprovação exige três assinaturas. Muda-se para "aprovação automática até R\$ 15 mil, notificação posterior" — reduz o custo de tempo para agir. Ou, ao contrário: aumentar o custo de algo indesejado — "toda reunião sem pauta escrita é cancelada automaticamente".

#### Destruir
Elimina um elemento, regra, código ou prática do agenciamento.

*Exemplo*: Eliminar uma reunião recorrente que ninguém sabe por que existe. Remover um campo obrigatório de um formulário que todo mundo preenche com "N/A". Apagar uma política que caiu em desuso mas ainda está no manual.

#### Estabilizar
Mantém uma configuração onde ela está funcionando bem.

*Exemplo*: Um time encontrou um ritmo de retrospectivas que funciona. A perturbação é proteger esse ritual de outras iniciativas, formalizar o horário, garantir que não será substituído por outra reunião.

### Ações de Sinal

#### Condicional
Cria um encadeamento entre eventos e ações — "se X acontecer, Y se inicia automaticamente".

*Exemplo*: Duas áreas (vendas e produto) têm conflito recorrente: vendas promete prazos que produto não consegue cumprir. A tensão já foi identificada, mas ninguém senta para conversar. A perturbação: a partir de agora, se a mesma reclamação de prazo aparecer em três reuniões consecutivas de liderança, a reunião de alinhamento entre as duas áreas é convocada automaticamente sem precisar de aprovação do diretor. O link é a intervenção, não a reunião em si.

#### Monitorar
Observação focada, geralmente em zonas de fronteira ou liminares, sem intervir.

*Exemplo*: Colocar um observador nas reuniões de handoff entre duas áreas que estão em conflito. Apenas registrar o que acontece, como um termômetro. Ou: criar um canal anônimo onde as pessoas podem sinalizar tensões sem medo.

#### Gatilho
As condições para agir estão presentes. O momento de agir chegou.

*Exemplo*: Um experimento de autonomia orçamentária foi desenhado e comunicado, mas não começa até que um gatilho ocorra — quando o time entregar três sprints consecutivos no prazo. O gatilho não é a ação; é a condição que autoriza a ação a começar. Quando acontece, age-se sem nova deliberação.

### Ações de Comunicação

#### Pesquisar
Investigar, fazer *sensemaking*, expandir opções.

*Exemplo*: Fazer uma rodada de entrevistas abertas com um time que está em estagnação. O objetivo não é resolver — é entender qual a qualidade da estagnação. Pode virar uma cartografia de tensões formal depois.

#### Solicitar
Colaboração, contato, permissão — frequentemente em zonas liminares onde a autoridade não está clara.

*Exemplo*: Um time quer mudar uma política de horários, mas não tem autoridade para isso. A perturbação é pedir ao diretor uma reunião de 30 minutos para apresentar a proposta e negociar uma exceção temporária.

#### Transparência
Torna visível algo que estava invisível, estimulando interações que antes não aconteciam.

*Exemplo*: Publicar o status de todas as aprovações pendentes e há quanto tempo estão esperando. Ou: abrir as decisões orçamentárias para o time. Cuidado: transparência pode redistribuir poder ou gerar efeito panóptico (pessoas se sentem vigiadas) — o contexto decide.

## Modalidades de perturbação

Nem toda perturbação atende o mesmo tipo de situação. Nomear a modalidade em que se opera nos ajuda a entender como vamos apresentar uma intervenção e como ela deve ser estruturada:

| Modalidade                         | Caráter                                                        | Reversibilidade                     | Domínio típico        |
| ---------------------------------- | -------------------------------------------------------------- | ----------------------------------- | --------------------- |
| **Ação imediata**                  | Necessidade de agir agora para conter, estabilizar ou proteger | Baixa (o custo de não agir é maior) | Caótico / Claro       |
| **Experimento seguro para falhar** | Escala pequena, o aprendizado vale o custo do fracasso         | Alta                                | Complexo              |
| **Projeto ordenado**               | Previsível, planejável, execução direta                        | Média                               | Claro / Complicado    |
| **Aposta de alto risco**           | Consequências significativas, baixa margem para erro           | Baixa                               | Complexo com urgência |

Uma perturbação pode transitar entre modalidades ao longo do tempo: um experimento seguro para falhar que se consolida vira projeto ordenado. Uma ação imediata que dá certo vira alvo de estabilização.

### Cadência multi-modal

Uma mesma tensão situada pode demandar intervenções em mais de uma modalidade, simultânea ou sequencialmente. Por exemplo: uma tensão de sobrecarga em um time pode exigir uma **ação imediata** (cancelar reuniões por 15 dias), um **experimento** (testar uma nova cadência de priorização) e um **projeto ordenado** (redesenhar o fluxo de entrada de demandas).

Da mesma forma, uma perturbação pode atender múltiplas tensões, e uma mesma tensão pode ser endereçada por múltiplos experimentos. A modalidade não é uma propriedade fixa da tensão, mas uma escolha de design sobre como intervir em cada faceta dela.

**Exemplo de ação imediata**: Um time está em burnout evidente — entregas atrasadas, pessoas adoecendo, clima de tensão. Identificou-se na cartografia que a sobrecarga vem de reuniões excessivas e demandas paralelas que ninguém interrompeu. A perturbação não precisa de piloto: cancela-se todas as reuniões não essenciais por 15 dias, efetivo imediatamente. Depois se avalia. Não é um experimento — é uma contenção. O custo de não fazer é maior que o custo de fazer.

## Canvas 1 — Ação Imediata

Para quando é preciso **conter, estabilizar ou destruir** algo imediatamente. O custo de não agir é maior que o custo de agir.

```
│ TENSÕES RELACIONADAS:                                        │
│                                                              │
├──────────────────────────────────────────────────────────────┤
│ HIPÓTESE                                                     │
│ SE [contivermos/destruirmos X agora]                         │
│ ENTÃO [o sistema alcança estabilidade Y]                     │
│ ANTES QUE [dano Z ocorra]                                    │
│                                                              │
├──────────────────────────────────────────────────────────────┤
│ O QUE ESTAMOS CONTENDO?                                      │
│ O que precisa parar, ser interrompido ou iniciado agora?    │
│                                                              │
│                                                              │
├──────────────────────────────────────────────────────────────┤
│ AÇÃO                                                         │
│ O que será feito, por quem, com que efeito imediato?         │
│                                                              │
│                                                              │
├──────────────────────────────────────────────────────────────┤
│ CRITÉRIO DE ESTABILIZAÇÃO                                    │
│ Como saberemos que a crise passou? O que precisa acontecer   │
│ para considerarmos que o sistema está minimamente estável?   │
│                                                              │
│                                                              │
├──────────────────────────────────────────────────────────────┤
│ PRÓXIMO PASSO                                                │
│ Depois de conter, o que vem? (experimento, projeto, aposta)  │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```
### Checklist da Ação Imediata

- [ ] A ação é realmente necessária agora? 
- [ ] A hipótese está no formato SE [ação] ENTÃO [estabilização] ANTES QUE [dano]?
- [ ] As tensões que motivaram a ação estão identificadas?
- [ ] O critério de estabilização é observável? 
- [ ] O próximo passo está definido? 
- [ ] A ação não cria mais dano do que o problema que está contendo?

### Checklist do Projeto Ordenado

- [ ] As tensões que motivam o projeto estão identificadas?
- [ ] O escopo é claro (inclui e exclui)?
- [ ] Os entregáveis são concretos e verificáveis?
- [ ] O cronograma é realista?
- [ ] As responsabilidades estão atribuídas?
- [ ] Os critérios de sucesso são observáveis?
- [ ] O domínio é realmente claro ou complicado? (se não for, trocar para experimento)

### Checklist da Aposta de Alto Risco

- [ ] As tensões que motivam a aposta estão identificadas?
- [ ] As alternativas de menor risco foram esgotadas?
- [ ] Os sinais vermelhos para abortar estão definidos e são observáveis?
- [ ] As pessoas impactadas sabem que o risco existe?
- [ ] Quem precisa autorizar foi consultado?
- [ ] Há um compromisso real de abortar se os sinais vermelhos aparecerem?
